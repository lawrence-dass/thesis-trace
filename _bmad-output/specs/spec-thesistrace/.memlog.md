- (note) WHY: A vision to realize (+ a builder's showcase). Diligence-driven retail investors (user zero = Lawrence) need proof, not opinions, before committing capital — company financial health, valuation, growth durability, and earnings integrity across four transparent lenses, every conclusion traceable to the filing line item. Deterministic forensic models (Piotroski/Altman/Beneish/Sloan) computed from raw XBRL, never an LLM-guessed number; the LLM explains and cites only. Secondary (~20%): demonstrate real financial-engineering/AI/full-stack skill to technical evaluators (D2).
- (capability) CAP-1 Company discovery & universe (FR-1, FR-2). intent: any visitor browses the Phase-1 Company Universe as cards and searches a ticker, no auth. success: landing page renders CP/QSR/OTEX/SHOP cards (name, ticker, last-updated) each linking to its overview; a ticker in-universe navigates to its page; out-of-universe returns an explicit 'not yet covered' — never a silent failure or fabricated result.
- (capability) CAP-2 Quality/Health lens computation — Piotroski F-Score + Altman Z-Score (FR-3, FR-4). intent: system computes both scores from XBRL Canonical Facts per company. success: all 9 Piotroski signals and all 5 Altman ratios individually computed and stored (not just aggregates); Altman market value of equity = FYE Tiingo close x EDGAR shares outstanding (never book-value substitute); financial-sector firms excluded, capital-intensive firms caveated; formula versioned; missing input = insufficient_data, never guessed.
- (capability) CAP-3 Quality/Health sub-signal display with provenance (FR-5). intent: user views each Piotroski signal (pass/fail) and Altman component (weight) with the exact values compared. success: each signal shows the specific compared values (e.g. 'ROA 2024 4.2% vs 2023 3.1% -> Pass') and links to its provenance record (source filing + line item).
- (capability) CAP-4 Integrity lens computation — Beneish M-Score + Sloan accruals ratio (FR-6, FR-7). intent: system computes both from Canonical Facts. success: all 8 Beneish indices individually computed/stored; Sloan computed via balance-sheet approach across two consecutive balance sheets and flagged against an explicitly stated published threshold; formulas versioned; financial-sector firms excluded from Beneish.
- (capability) CAP-5 Integrity sub-signal display, provenance & data-quality flags (FR-8). intent: user views each Integrity signal's inputs and source filing line item. success: same provenance-linking as CAP-3; an accounting-identity validation failure (e.g. balance sheet doesn't balance) surfaces as an explicit data-quality warning via the single data_quality_issues contract, never silently hidden.
- (capability) CAP-6 Company overview & Verdict (FR-9, FR-10). intent: each company has an overview page showing a transparent Verdict plus all currently-live lens scores, with in-page expandable sub-factor breakdowns. success: Verdict is a side-by-side juxtaposition of each live model's own published, cited threshold classification (Piotroski Strong 8-9/Weak 0-1/2-7 Middle; Altman Safe/Grey/Distress; Beneish >-1.78; Sloan spec threshold) — never a blended number, never LLM-invented, band labels computed backend; page states which lenses are live vs pending; individual scores stay visible/clickable; expansion is in-page accordion linking onward to methodology.
- (capability) CAP-7 Methodology drill-down (FR-11). intent: each Deterministic Score has a dedicated methodology page. success: page states the formula, the exact XBRL concepts mapped to each input, the formula version in use, and links to the model's original academic source.
- (capability) CAP-8 AI score explanation — deterministic-first, cited (FR-12). intent: user requests a plain-language explanation of any score or the Verdict. success: text is rendered by a deterministic template from already-computed scores/facts; the LLM (default Claude Haiku 4.5, env-keyed, narrow direct wrapper, no LangGraph) only polishes already-correct text, never originates a claim/number/citation; every explanation carries inline citations to provenance records; an ungroundable statement is omitted.
- (capability) CAP-9 Comparison (FR-13, FR-14). intent: user adds 2-4 viewed companies to an in-session comparison and views their Verdicts and live lens scores in parallel columns. success: set is browser-session-only (no server persistence, no auth); min 2 / max 4; view shows exactly the lenses live in the current phase; differences beyond a stated threshold are visually highlighted.
- (capability) CAP-10 Cited filing Q&A (FR-15, Phase 2 directional). intent: user asks a free-text question about a company's filings and gets a cited, evidence-checked answer or explicit 'insufficient evidence'. success: every claim carries a citation to a filing passage; ungroundable claims omitted/flagged; orchestrated via LangGraph with a real self-verifying citation loop (D7). Detailed at Phase-2 architecture.
- (capability) CAP-11 Value lens (FR-16, Phase 2 directional). intent: system computes valuation metrics (DCF, reverse DCF, Graham Number, yields, multiples) with stated assumptions. success: no valuation shown as a bare point estimate — assumptions + sensitivity range always accompany it; missing inputs follow the insufficient_data pattern.
- (capability) CAP-12 Growth lens (FR-17, Phase 2 directional). intent: system computes growth-trajectory metrics across available fiscal years and presents historical trends via custom visualizations. success: trend depth honestly reflects each company's available history (states coverage window); missing periods follow insufficient_data.
- (capability) CAP-13 Thesis journal & re-verification (FR-18, Phase 2/3 directional). intent: user saves a written thesis auto-attached to live lens scores/facts; on return the system produces a per-claim Thesis Diff against current values. success: persisted browser-locally (no auth); diff is per-claim/per-metric, not a single 'something changed' flag.
- (capability) CAP-14 Notifications & deep-research requests (FR-19, FR-20, FR-21, Phase 3 directional). intent: email-only (no account) notifications for thesis re-verification availability and deep-research fulfillment. success: thesis-diff email fires once per new filing per thesis with a change summary; deep-research request queued with a stated SLA and answered with citations (or explicit insufficient-evidence) via a real transactional email provider.
- (constraint) Deterministic/LLM boundary (inviolable, D5/D7): deterministic services compute every number, score, threshold, and verdict input; the LLM only explains/summarizes/answers, always cited, never canonical. The LLM-generated 0-100 health score is permanently rejected.
- (constraint) Architecture is binding: the 21 ADs in the adopted ARCHITECTURE-SPINE.md companion govern all build decisions — batch-pipeline-write / read-only-query CQRS split (AD-1), immutable raw store + versioned canonicalization (AD-2/AD-3), versioned formula specs with one shared rounding engine (AD-5/AD-15), append-only score_runs (AD-6), tri-state signal status (AD-16), single data_quality_issues contract (AD-17), canonical score_results shape (AD-18), provenance as a first-class invariant (AD-19), sector-scope applicability state (AD-20).
- (constraint) Data sources (D7 exception): SEC EDGAR Company Facts API primary + Inline XBRL fallback; Tiingo free tier is the ONLY additional provider, solely for Altman's FYE closing price. No other data provider in Phase 1. EDGAR access discipline: identifying User-Agent, <=10 req/s, cached/retried/idempotent/replayable (AD-9).
- (constraint) Phase-1 Company Universe is fixed and US-GAAP-only: CP, QSR, OTEX, SHOP — all 10-K / us-gaap, non-financial (D6). IFRS/40-F filers are excluded from the Piotroski/Altman/Beneish/Sloan formula set (they would break the formulas as specified).
- (constraint) No end-user auth in Phase 1 (D4): public, read-only. Admin/recompute is operator-only via a shared-secret header (AD-10). Designs must not foreclose a future B2C pivot (clean auth seams) but must not build auth/multi-tenancy/billing early.
- (constraint) Cost ceiling ~$25/month total (hosting + data + LLM). Current fixed cost ~$8-10/mo: Render web+cron (AD-13) + Supabase free Postgres 17 + Tiingo free + Vercel free — leaving headroom for LLM spend. Supabase 7-day auto-pause is kept alive by the daily batch ingestion job.
- (constraint) Phase-1 LLM is a plain, direct API wrapper — NO LangChain/LangGraph (D7); default Anthropic Claude Haiku 4.5, key via env var, provider swappable, out of the numeric loop (AD-21). LangGraph is reserved for the Phase-2 filing-Q&A self-verifying citation loop only.
- (constraint) Visualizations are custom-built (e.g. Recharts/visx) over ThesisTrace's own computed data — no off-the-shelf charting widgets (TradingView explicitly excluded, D7), differentiating from the sibling equipulse project.
- (constraint) Web-only, responsive; no native mobile app (deliberate product-scope choice, research-session use case is desktop-leaning). Stack pinned: Next.js 16.2.x/React 19.2.x on Vercel; FastAPI 0.139.x + Pydantic 2.13.x + SQLAlchemy 2.0.51 async / Python 3.12-3.13 on Render; Postgres 17 (Supabase).
- (constraint) SM-1 correctness is the Phase-1 quality bar: a golden-dataset regression harness (all four models' outputs vs hand-verified/published values) is in Phase-1 scope; only its CI automation is Phase-4-deferred.
- (decision) NON-GOALS (PRD section 5, permanent unless noted): never investment advice; no LLM-generated canonical scores ever; no brokerage integration / trade execution / fee comparison; no full market coverage (no S&P 500 / broad TSX-TSXV); no user accounts or passwords beyond Phase-3 email-only capture; no off-the-shelf charting widgets; no IFRS-reporting companies in the current formula set; no native mobile app; no broad news/sentiment via paid providers; no sector heatmap or draggable/resizable dashboard; no WCAG compliance target for v1; no formal performance/uptime/security NFR targets for v1 (correctness is the bar); no SEDAR+/TSX-only non-EDGAR companies.
- (decision) SUCCESS SIGNAL (primary): SM-1 — 100% of Piotroski/Altman/Beneish/Sloan scores for the Phase-1 universe match a hand-verified or published golden dataset, enforced by regression tests. SM-2 — ThesisTrace informs at least one real investment decision by user zero within 3 months of Phase-1 launch. (Secondary SM-3 engineering-showcase, SM-4 stickiness; counter-metrics SM-C1 don't chase universe breadth before correctness, SM-C2 don't optimize notification volume.)
- (question) OQ (Phase-1, from PRD OQ1): golden-dataset sourcing — where do hand-verified/published Piotroski/Altman/Beneish/Sloan scores for CP/QSR/OTEX/SHOP come from, and how is the set kept current as filings arrive? Blocks SM-1.
- (question) OQ (Phase-1, from PRD OQ3): not-investment-advice disclaimer — exact wording, placement (every page? footer? first-visit notice?), and whether Canada-specific legal review is warranted.
