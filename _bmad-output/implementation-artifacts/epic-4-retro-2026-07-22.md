## What went well

- **End-to-end pipeline proven on real data**, not just fixtures — EDGAR ingestion → canonicalization → scoring → API → UI, for a real universe of 4 companies.
- **The branch-per-session + PR workflow (CLAUDE.md), adopted after the parallel-session incident, held.** PRs #2 through #8 show no repeat of the git-divergence collision that motivated it.
- **The `TEST_DATABASE_URL` bug (PR #4) was caught and fixed proactively**, before it caused data loss in ongoing work, by someone getting local dev running and reading the fixture teardown carefully.
- **Design system (PR #6) reused a real domain model** (the tri-state signal palette maps directly onto the backend's `SignalStatus`/`Applicability` enums) rather than inventing arbitrary UI colors — this paid off directly in this session's gauge work, which reused the same tokens with zero new palette decisions.

## What didn't go well

**The dominant pattern: real bugs were found *after* stories were marked `done`, not during — and only when someone deliberately re-verified against real data instead of trusting the green test suite.**

1. **Golden-dataset harness (Stories 1.9, 2.6) is marked `done` but isn't.** `phase1_golden.yaml` still carries its own `⚠️ PLACEHOLDER VALUES — NOT YET AUTHORITATIVE` header; CP/QSR/OTEX have zero real golden coverage. SM-1 ("100% of scores match a hand-verified golden dataset") has never actually been met — it looked done because a synthetic fixture passed, not because real values were hand-verified. Investigation is currently blocked in cloud sessions without Docker/Postgres/EDGAR network access (see `HANDOFF.md`'s 2026-07-22 session note).
2. **`shares_outstanding` silently resolved to the wrong fiscal year for essentially every real company** (`dei:EntityCommonStockSharesOutstanding` is filing-dated, not FYE-dated) — missed by the shipped test suite because its fixture used unrealistic dates. Found by a second, independent Reviewer Gate pass, not by CI.
3. **`accession_number` was derived from the wrong source** (`Filing.fiscal_year` instead of `CanonicalFact`), breaking a foreign-key constraint the first time a real filing's data spanned fiscal years (SHOP FY2023).
4. **`raw_facts.concept` was `VARCHAR(128)`**, too narrow for a real 140-character XBRL extension tag — hit immediately on live CP data.
5. **CP, QSR, and OTEX had no CIK wired into `pipeline/universe.py`** — 3 of the 4 "done" companies never actually ran through the live pipeline until PR #5, despite Epic 1/2 stories claiming full-universe coverage.
6. **`pipeline/run.py` never called Tiingo** — Altman couldn't compute via a live run for anyone, including SHOP, until PR #5.
7. **Beneish never computes a value for any company, ever** — 3 of its 8 required concepts have zero canonicalized mappings for any of the 4 companies. Still open.
8. **SHOP's Altman never resolves to a non-null value**, root cause not yet diagnosed. Still open.
9. **`bandTone()` (PR #6) has no keyword match for Beneish's actual band labels** ("Manipulation risk flagged" / "No manipulation flag") — every Beneish badge has rendered neutral gray in production since PR #6, regardless of the real result. Found and fixed in this session while building the score-teaching gauge (below) — the bug had been live and unnoticed because nothing before now rendered Beneish's band color in a way that made the miscoloring obvious.

None of items 2–9 were caught by the 48-test green suite. All were found by deliberately re-verifying against real EDGAR/Tiingo data or a second independent review pass — work that happened late and reactively, not as part of any story's Definition of Done.

**Secondary observation:** Story 1.1's own file still shows `Status: ready-for-review`, while `sprint-status.yaml` says `done` — the per-story files and sprint-status already drifted after the very first story. Not a functional bug, but worth knowing if anyone later trusts story files as source of truth.

## Key insight

"48 green tests" and "story marked done" measured *logic correctness against synthetic fixtures*, not *correctness against real filings*. The synthetic fixtures were realistic enough to validate formulas but not realistic enough to catch date semantics, missing-CIK wiring, incomplete concept mappings, or a code path that was never actually exercised end-to-end. Every bug above needed a live-data pass to surface.
